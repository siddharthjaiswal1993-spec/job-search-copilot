# Outcome Model — Job Search Copilot

---

## The Value Hypothesis

Job Search Copilot creates value by making apply/pass decisions faster and higher quality, and by reducing the time spent generating application materials. The outcome is a more focused, more efficient job search process with higher signal-to-noise ratio.

---

## User Outcome Metrics

| Metric | Target | Why It Matters |
|---|---|---|
| Time to qualify a role | <5 minutes (AI score + rationale) vs. 20+ minutes manual | The core time-savings value |
| Daily active role reviews | 5–10 per session | Indicates the daily digest is surfacing relevant roles |
| Application pack generation rate (high-fit roles) | >80% of Apply-recommended roles get a pack | Measures follow-through conversion |
| Time per application pack | <3 minutes (generate → review → edit) | Measures generation quality |

---

## AI Quality Metrics

| Metric | Target | Why It Matters |
|---|---|---|
| Scoring consistency | <10% variance across re-scores of same role | Inconsistent scoring destroys confidence in recommendations |
| Apply recommendation quality | >70% of Apply-rated roles rated as genuinely strong fit upon review | Precision on Apply recommendations is critical |
| Application pack usefulness | >80% of generated assets used (possibly edited) | Generated assets should be starting points, not discards |
| Sub-score explainability | All 6 sub-scores include written rationale | Unexplained scores are not actionable |

---

## What Good Looks Like

A job search using this tool looks like:
- Daily digest surfaces 5–8 relevant new roles each morning
- 80% are qualified (Apply/Maybe/Skip) in under 30 minutes total
- High-fit roles have complete application packs within the same session
- Pipeline kanban accurately reflects current status across all active applications
- Weekly review of score distribution confirms the model is calibrated to actual preferences

---

## The Metric That Matters Most

**Pack generation rate for Apply-rated roles.**

If a user rates a role Apply but does not generate an application pack, there is friction in the workflow or the pack quality is not trusted. That gap between "this looks good" and "I'm actually applying" is the primary drop-off to measure and reduce.
